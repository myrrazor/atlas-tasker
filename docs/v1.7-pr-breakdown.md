# Atlas Tasker v1.7 PR Breakdown

The v1.7 train is stacked:

```text
PR-701 -> PR-702 -> PR-703 -> PR-704 -> PR-705 -> PR-706 -> PR-707 -> PR-708
```

## PR-701 — Contracts, Trust Model, Canonicalization, Design Freeze

Freeze the security contracts, command surface, storage paths, sync/local-only boundaries, event families, JSON kinds, and docs. Add command scaffolding and contract tests only. No signing or governance engine behavior lands here.

Acceptance:

- v1.7 base metadata is recorded.
- v1.6.1 hosted proof gap is documented.
- key lifecycle, verification states, canonicalization, governance precedence, quorum semantics, redaction preview binding, audit snapshot semantics, backup limits, goal output contract, and MCP alignment are frozen.
- command inventory tests prove all v1.7 top-level families and planned subcommands exist.

## PR-702 — Workspace Keys, Trust Store, Signing Primitives

Implement key storage, public/private key lifecycle, Unix `0600` private-key checks, public-key import/export, local trust bindings, Ed25519 sign/verify primitives, `tracker key`, `tracker trust`, inspect/doctor trust health, and leakage protections.

Acceptance:

- private keys never sync, export, log, or print
- imported public keys do not become trusted automatically
- revoked keys cannot sign new artifacts
- rotated/expired keys verify old artifacts with warning states

## PR-703 — Signed Bundles, Sync Provenance, Backup Manifests

Add optional signatures to export bundles, sync publications, and backup manifests. Add pure `tracker verify` commands for bundle, sync publication, backup, handoff, audit, and goal artifacts. Unsigned artifacts remain compatible only where policy allows.

Acceptance:

- signed/tampered/untrusted/revoked/missing-signature cases return shared verification states
- sync publication signing excludes local-only paths
- verification does not mutate by default

## PR-704 — Governance, Quorum, Separation-Of-Duties, Protected Actions

Implement governance pack storage, `governance explain`, `governance simulate`, protected-action enforcement, quorum, separation-of-duties, owner override events, and old-path bypass tests.

Acceptance:

- protected actions call one governance evaluator
- deny wins over allow
- owner override is itself protected and evented
- quorum is evaluated at action time by root collaborator identity

## PR-705 — Classification, Redaction, Data-Sharing Controls

Implement classification labels and inheritance, redaction rules, preview records, redacted export/sync/audit/backup, reimport warnings for redacted artifacts, and secret hygiene across outputs.

Acceptance:

- default classification is `internal`
- higher sensitivity wins
- stale or mismatched previews cannot authorize redacted artifacts
- restricted content is absent from redacted artifacts

## PR-706 — Audit Reports, Signed Evidence, Decision Provenance

Implement audit report/list/view/export/verify, signed approvals/handoffs/evidence packets where policy requires them, policy decision provenance, deterministic snapshots, and tamper detection.

Acceptance:

- audit reports include policy/trust snapshot hashes and included artifact hashes
- verification checks packet snapshot semantics
- policy explanations include actor, collaborator, membership, trust, signatures, quorum, and classification inputs

## PR-707 — Backup, Disaster Recovery, Admin Diagnostics, Goal Manifests

Implement backup create/list/view/verify, restore-plan/apply, recovery drill, admin security diagnostics, then `tracker goal brief`, `tracker goal manifest`, and `tracker goal verify`.

Acceptance:

- backups exclude private keys by default
- restore planning is side-effect free
- restore apply uses write lock/journal discipline
- goal outputs are stable, redaction-aware, provider-neutral markdown and JSON

## PR-708 — Proving, Adversarial Security Tests, Docs, Release Evidence

Add adversarial tests, fuzzing, cross-surface parity, documentation polish, release rehearsal, `govulncheck ./...` when available, SBOM decision/proof, hosted checksums/attestations, and final `TEST_STDOUT.log`.

Acceptance:

- `go test ./... 2>&1 | tee TEST_STDOUT.log`
- `go vet ./...`
- `sh scripts/stability-smoke.sh`
- `VERSION=v1.7.0-rc1 ./scripts/release-rehearsal.sh`
- hosted prerelease proof or explicit no-ship decision

# Atlas Tasker v1.3 Implementation Plan

Atlas Tasker v1.3 hardens the shipped v1.2 local-first core before layering on automation, SCM workflows, subscriptions, bulk ops, and TUI expansion.

## Priorities

1. Reliability foundations: workspace write locking, shared clock semantics, explicit error taxonomy, stable JSON contracts.
2. Durability and repair: mutation journal, atomic rebuild/swap, idempotent repair.
3. Operator workflows: safe automation, notifier transport, git local integration, saved views, watchers, bulk ops.
4. Interface expansion: TUI panels only after service contracts stabilize.
5. Release proof: chaos, benchmarks, prerelease rehearsal, operator docs.

## Reuse

v1.3 builds on the existing v1.2 layers:
- `ActionService` and `QueryService`
- markdown snapshots under `projects/`
- append-only JSONL events under `.tracker/events/`
- SQLite projection under `.tracker/index.sqlite`
- CLI, shell, TUI, notifier, and release/install flows

## Repo-aware deviation

The implementation plan asked for `feat/* -> testing` on top of `dev`, but this repo only has `main` and the existing stacked `codex/pr-*` branches. v1.3 continues the established stacked branch model and records that deviation in the v1.3 decision log instead of inventing new long-lived branches inside the train.

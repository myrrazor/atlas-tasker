# Atlas Tasker v1.2 Decision Log

This file records v1.2 design decisions so later PRs can change them deliberately instead of by accident.

## V12-001

1. **Decision ID:** V12-001
2. **Date:** 2026-03-22
3. **Question:** Should v1.2 keep the v1 CLI as the source of truth?
4. **Options Considered:**
   - CLI remains canonical and TUI is additive.
   - TUI becomes the new source of truth.
5. **Chosen Option:** CLI remains canonical; TUI is additive.
6. **Why We Chose It:** It preserves v1 compatibility and avoids duplicating business logic across interfaces.
7. **Confidence:** high
8. **Revisit Trigger:** A later major version intentionally pivots to TUI-first workflows.
9. **Affected PRs/Files:** PR-201..PR-211; `internal/service/*`, `internal/cli/*`, `internal/tui/*`.

## V12-002

1. **Decision ID:** V12-002
2. **Date:** 2026-03-22
3. **Question:** How should v1.2 represent policy, review, and lease state in persisted data?
4. **Options Considered:**
   - Extend current ticket/project contracts and bump schema version.
   - Keep v1 schema and store v1.2 state only in SQLite.
5. **Chosen Option:** Extend contracts and bump schema version to `2`.
6. **Why We Chose It:** Markdown remains the readable snapshot source and must include the new state.
7. **Confidence:** high
8. **Revisit Trigger:** Snapshot verbosity becomes unmanageable.
9. **Affected PRs/Files:** PR-201..PR-205; `internal/contracts/*`, `internal/storage/markdown/*`, `internal/storage/sqlite/*`.

## V12-003

1. **Decision ID:** V12-003
2. **Date:** 2026-03-22
3. **Question:** What terminal stack should the new full-screen interface use?
4. **Options Considered:**
   - Bubble Tea + Bubbles + Lip Gloss.
   - Custom terminal renderer stack.
   - Another form/UI framework on top of Bubble Tea.
5. **Chosen Option:** Bubble Tea + Bubbles + Lip Gloss only.
6. **Why We Chose It:** It matches the planned UX without introducing a second interaction framework.
7. **Confidence:** high
8. **Revisit Trigger:** Bubble Tea proves insufficient for key workflows.
9. **Affected PRs/Files:** PR-206..PR-208; `internal/tui/*`.

## V12-004

1. **Decision ID:** V12-004
2. **Date:** 2026-03-22
3. **Question:** What is the default lease TTL for upgraded and new workspaces?
4. **Options Considered:**
   - 30 minutes.
   - 60 minutes.
   - 120 minutes.
5. **Chosen Option:** 60 minutes.
6. **Why We Chose It:** It gives agents enough time to work without making stale claims linger for too long.
7. **Confidence:** medium
8. **Revisit Trigger:** Real usage shows claims expiring too early or too late.
9. **Affected PRs/Files:** PR-201..PR-204; contracts, queue logic, doctor/sweep.

## V12-005

1. **Decision ID:** V12-005
2. **Date:** 2026-03-22
3. **Question:** How should v1.2 handle v1 workspace compatibility?
4. **Options Considered:**
   - Require a one-shot migration command before use.
   - Load missing fields with defaults and upgrade lazily on write.
5. **Chosen Option:** Load defaults and upgrade lazily on write.
6. **Why We Chose It:** It keeps existing workspaces usable immediately and avoids a risky hard cutover.
7. **Confidence:** high
8. **Revisit Trigger:** Lazy upgrades create ambiguity or performance issues.
9. **Affected PRs/Files:** PR-201, PR-211; codecs, rebuild, fixtures, docs.

## V12-007

1. **Decision ID:** V12-007
2. **Date:** 2026-03-22
3. **Question:** Should v1.2 vendor dependencies or add a private mirror?
4. **Options Considered:**
   - vendor now.
   - private module mirror now.
   - keep normal Go modules for v1.2.
5. **Chosen Option:** keep normal Go modules for v1.2.
6. **Why We Chose It:** Release packaging will ship binaries, and vendoring would mostly bloat the repo during active iteration.
7. **Confidence:** medium
8. **Revisit Trigger:** Air-gapped builds or supply-chain requirements harden.
9. **Affected PRs/Files:** PR-211 docs/release process.

## V12-008

1. **Decision ID:** V12-008
2. **Date:** 2026-03-22
3. **Question:** How should CLI and TUI access read models?
4. **Options Considered:**
   - direct SQLite queries from each interface.

5. **Chosen Option:** shared query/view-model services.

9. **Affected PRs/Files:** PR-201..PR-208; `internal/service/*`, `internal/cli/*`, `internal/tui/*`.

## V12-009

1. **Decision ID:** V12-009
2. **Date:** 2026-03-22
3. **Question:** How should actor resolution work for queue/claim workflows?
4. **Options Considered:**
   - explicit flag only.
   - explicit flag, env var, config default fallback.
5. **Chosen Option:** explicit `--actor`, then `TRACKER_ACTOR`, then `actor.default`.
6. **Why We Chose It:** It keeps automation explicit while making repeated local use less annoying.
7. **Confidence:** high
8. **Revisit Trigger:** Authentication or multi-user identity is introduced.
9. **Affected PRs/Files:** PR-201..PR-203; config, services, CLI.

## V12-015

1. **Decision ID:** V12-015
2. **Date:** 2026-03-22
3. **Question:** What is the release bar for terminal accessibility in the TUI?
4. **Options Considered:**
   - best effort only.

7. **Confidence:** high
8. **Revisit Trigger:** None without a product strategy change.

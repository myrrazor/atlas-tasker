## Summary
v1.5 is the delivery and productization layer on top of the integrated v1.4.1 train.
It keeps the local-first architecture intact:
- markdown snapshots remain readable current state
- JSONL events remain immutable history
- SQLite remains a rebuildable projection only
- shared services remain canonical for CLI, shell, and TUI
- replay, repair, reindex, archive planning, and import preview never recreate live side effects

## Mainline scope

2. checks and merge readiness
3. permissions and trust model 2.0
4. import / export / migration
5. retention / archival / compaction
6. dashboards / timelines / TUI scale
7. release / distribution proof

## Hard rules
- provider state is enrichment, never canonical truth
- GitHub is optional
- no daemon scheduler
- no remote sync or multi-user server mode
- no MCP in the mainline
- no TUI service bypasses
- no replay/repair/provider/archive flow may recreate worktrees, runtime side effects, provider writes, or external review state

# Atlas Tasker v1.8 Implementation Plan

v1.8 is the release-candidate polish train for Atlas Tasker. The product already has the core local-first tracker, agent run lifecycle, gates, evidence, handoffs, sync, MCP RC adapter, and v1.7 trust/governance/security surfaces. v1.8 makes that work readable, installable, demoable, and provable.

## Goals

- Make the GitHub README and docs good enough for a new technical user to understand Atlas in minutes.
- Polish terminal, Markdown, and TUI output without changing storage or workflow architecture.
- Make Codex, Claude Code, generic coding agents, and MCP users understand the safe Atlas workflow.
- Provide demo workspaces, transcripts, scripts, and examples that can be regenerated locally.
- Close release proof around `v1.8.0-rc1`: checksums, attestations, SBOM, vulnerability scan, packaged smoke, and explicit ship/no-ship evidence.

## Non-Goals

- No hosted server, SaaS auth, daemon scheduler, CRDT rewrite, plugin marketplace, mandatory MCP flow, new database, or RAG system.
- No redesign of Markdown snapshots, JSONL events, SQLite projection, ActionService/QueryService, CLI/shell/TUI/MCP client boundaries, or v1.7 signing/governance/redaction contracts.
- No broad provider automation beyond docs, examples, and existing command surfaces.

## Review And Gate Protocol

Before each PR, attempt a GStack/Codex review of that PR's plan or current stack state. After each PR, run deterministic checks first, then a GStack/Codex review of the diff. Record unavailable review tooling in this decision log instead of silently skipping it.

Required local gates for every PR:

- `git diff --check`
- `go test ./...`
- `go vet ./...`
- relevant targeted tests for touched packages/scripts/docs

Final gates for PR-809:

- `go test -count=1 ./... 2>&1 | tee TEST_STDOUT.log`
- `sh scripts/stability-smoke.sh`
- `VERSION=v1.8.0-rc1 ./scripts/release-rehearsal.sh`
- `go run golang.org/x/vuln/cmd/govulncheck@v1.3.0 ./...`
- CycloneDX SBOM generation through pinned `go run github.com/CycloneDX/cyclonedx-gomod/cmd/cyclonedx-gomod@v1.9.0 app -main cmd/tracker`
- hosted checksum, attestation, install, and packaged smoke proof when assets exist

## Release Rule

v1.8 can complete local RC proof without being declared ship-ready. Public release sign-off requires hosted prerelease assets and evidence. If hosted assets are unavailable, PR-809 must record a no-ship release decision with the exact blocked gate.

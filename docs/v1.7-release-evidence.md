## Hosted Proof

Blocked:

- command: `sh scripts/preflight-release-proof.sh`
- result: `preflight_status=blocked`
- reason code: `workflow_permissions_unreadable`
- GitHub API status: `403`

This session can verify the release workflow file and local rehearsal, but it cannot verify repository Actions workflow-permission settings through the current token. Do not call v1.7 release-ready until a release actor with the needed GitHub permission completes hosted prerelease proof.

Required hosted gate:

1. Run `sh scripts/preflight-release-proof.sh` with a token that can read repository Actions workflow permissions.
2. Create the selected prerelease tag, such as `v1.7.0-rc1` or a later candidate tag if more fixes land.
3. Wait for the GitHub release workflow to publish archives and `checksums.txt`.
4. Download at least one published archive.
5. Run `VERSION=<selected-candidate-version> ./scripts/verify-release.sh <archive-path>` with attestation verification enabled.
6. Record the archive name, checksum result, and `gh attestation verify` result here before promoting anything to a stable release.

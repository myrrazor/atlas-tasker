# Atlas Tasker v1.7 Implementation Plan

v1.7 is the trust, governance, and production-security train. It builds on the completed local-first collaboration system without changing Atlas' core architecture: markdown remains current state, JSONL events remain history, SQLite remains rebuildable projection, `ActionService` remains the write path, and `QueryService` remains the read path.

The train starts from `origin/main@e0083de1cfd8007d1331e211723f2c7c47f45ede`. v1.6.1 local release proof is green, but hosted prerelease proof is still blocked in `docs/v1.6.1-release-evidence.md`. v1.7 can begin from this base; it cannot be called release-ready until PR-708 completes hosted proof or records a no-ship decision.

## Scope

v1.7 adds:

- local signing identities, public key records, key revocation, and rotation contracts
- local trust bindings that do not silently sync into authority
- Ed25519 signing and shared verification states
- signed bundles, sync publications, backups, audit reports, approvals, handoffs, evidence packets, and goal manifests where policy requires them
- governance policy packs with quorum and separation-of-duties
- protected-action enforcement through the existing permission/collaborator/membership model
- classification labels, redaction rules, and preview-bound redacted outputs
- audit reports and verifiable audit packets
- backup, restore planning, recovery drills, and admin diagnostics
- read-only goal briefs/manifests for Codex `/goal`, Claude Code, and other agents
- production security docs and release proof

## Non-Goals

v1.7 does not add hosted server mode, SSO, encrypted storage, OS sandboxing, CRDT sync, daemon scheduling, provider-rule automation, MCP-first workflows, private-key export, or Codex app-server integration.

## Architecture Rules

- Signing proves artifact authenticity relative to a key. It does not prove authority.
- Authority is evaluated by governance plus the existing permission profile, collaborator, and membership model.
- Public key records and revocations may sync. Private keys and local trust decisions stay local-only by default.
- Verification, status, replay, reindex, repair, and doctor paths are side-effect free by default.
- v1.7 contracts use explicit schema versions and compatibility tests; existing event UID and Lamport-clock semantics stay stable.
- Redaction operates on structured fields wherever Atlas owns the structure.
- Audit reports are derived snapshot artifacts, not canonical truth.
- Backup restore is preview-first and never recreates provider state, worktrees, runtime dirs, notifiers, MCP approvals, or remote-side state.
- Goal briefs/manifests are read-only derived outputs and never control Codex or Claude Code.

## Review Workflow

Before PR-701 implementation, run GStack plan review and record accepted changes here and in `docs/v1.7-decision-log.md`. After each PR, run deterministic tests first, then GStack `/review`, Codex review, and Claude Code high-effort/ultrareview if available. Model review is supplemental; deterministic gates decide whether the next PR can start.

## Done Criteria

v1.7 is complete when signed artifacts can be generated and verified, trusted/untrusted/revoked signer states are consistent, governance blocks protected-action bypasses, redaction previews bind to source state, audit packets verify, backup drills pass without local side effects, goal manifests are stable and redaction-aware, and release proof includes tests, rehearsal, vulnerability/SBOM decisions, checksums, and attestations or a no-ship decision.

## Summary
v1.6 is the collaboration and sync release on top of the integrated v1.5 tip.
It keeps the local-first architecture intact:
- markdown snapshots remain readable canonical state
- JSONL events remain immutable history
- SQLite remains rebuildable projection only
- shared services remain canonical for CLI, shell, and TUI
- replay, repair, reindex, sync preview, and bundle verification never recreate live side effects

## Mainline scope
1. collaborator identity and project membership
2. git-backed Atlas sync remotes
3. offline bundle exchange fallback
4. sync-safe immutable UIDs for syncable entities
5. deterministic reconciliation and explicit conflict records
6. collaborator-aware inbox, mentions, approvals, and orchestration
7. provider/team mapping and CODEOWNERS/rules scaffolding
8. collaboration dashboard/timeline/TUI surfaces
9. proving, upgrade, and packaged multi-workspace release smoke

## Hard rules
- provider state is enrichment, never canonical truth
- no central always-on server
- no daemon scheduler
- no CRDT rewrite
- no MCP in the mainline
- no replay/repair/provider/sync/archive flow may recreate worktrees, runtime side effects, provider writes, or external review state
- multi-workspace sync is blocked until deterministic UID stamping is complete locally

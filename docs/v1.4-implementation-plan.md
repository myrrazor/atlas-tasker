# Atlas Tasker v1.4 Implementation Plan

Atlas Tasker v1.4 turns the existing local-first tracker into a local control plane for parallel AI coding work.

Implementation base:
- `codex/pr-314-stabilization-proof`

Core invariants carried forward:
- markdown snapshots remain readable current state
- JSONL events remain immutable history
- SQLite remains rebuildable projection only
- shared services remain canonical for reads and writes
- replay, repair, and reindex stay side-effect free beyond projection/journal reconciliation

Mainline PR train:
1. PR-401 contracts, migrations, and design freeze
2. PR-402 agent registry and capability matching
3. PR-403 run lifecycle and worktree manager
4. PR-404 dispatch engine and execution blueprints
5. PR-405 evidence bundles and handoff packets
6. PR-406 approval gates and human inbox
7. PR-407 integration generators and launch manifests
8. PR-408 TUI operator console expansion
9. PR-409 proving, upgrade, performance, and release docs

Hard constraints:
- no storage redesign
- git/worktrees are execution isolation, not source of truth
- no daemonized scheduler
- no remote sync
- no MCP in the mainline train
- no multi-user scope expansion
- no provider session supervision

Review flow:
- Claude Code Opus 4.5 effort-high `/autoplan` before each PR
- Claude Code Opus 4.5 effort-high review on each bounded diff
- Gstack `/review` on each PR branch
- Claude QA for PR-403, PR-404, PR-406, PR-408, PR-409, falling back to Gstack `/qa-only` then `/qa`

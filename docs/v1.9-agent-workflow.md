# v1.9 Agent Workflow

v1.9 adds a narrow coordination layer for coding agents. Atlas still does not host agents or run a scheduler by default; it exposes clearer work queues and event-driven wake-up records.

## Agent Work Queues

Use these reads first:

```bash
tracker agent available <agent-id> --json
tracker agent pending <agent-id> --json
```

`available` lists work the agent can act on now. `pending` lists relevant work that is blocked or waiting, with reason codes such as `dependency_blocked`, `waiting_for_review`, `claimed_by_other`, `policy_blocked`, and `agent_at_capacity`.

## Wake-Ups

When a ticket reaches `done`, Atlas checks tickets blocked by that ticket. If an assigned agent ticket is now available, Atlas records an `agent.work_available` event and a local wake-up record:

```bash
tracker agent wakeups list <agent-id> --json
tracker agent wakeups view <wakeup-id> --json
tracker agent wakeups ack <wakeup-id> --actor agent:<agent-id> --reason "picked up"
```

Only `done` unblocks dependencies. `canceled` does not. `human:owner` can intentionally override unresolved dependencies with `--override-deps --reason <TEXT>` on unsafe ticket progress commands; Atlas records the unresolved blockers in the mutation history.

## Auto Mode

Default mode is notify-only:

```bash
tracker agent auto status <agent-id>
```

Owners can enable command mode with argv items:

```bash
tracker agent auto set <agent-id> \
  --mode command \
  --argv /absolute/path/to/agent-runner \
  --argv "{ticket_id}" \
  --argv "{wakeup_id}" \
  --actor human:owner \
  --reason "enable local wake-up runner"
```

Atlas never stores shell strings for command mode. Direct shell interpreters such as `sh`, `bash`, `zsh`, `fish`, `cmd`, `powershell`, and `pwsh` are rejected.

## Surface Parity

| Capability | CLI | Slash shell | TUI | MCP |
|---|---:|---:|---:|---:|
| Agent available work | yes | yes | Queues tab | `atlas.agent.available` |
| Agent pending work | yes | yes | Queues tab | `atlas.agent.pending` |
| Wake-up list/view/ack | yes | yes | Ops tab list | deferred |
| Agent auto status/set/off | yes | yes | Ops tab status only | deferred |
| Skill pack install | yes | yes | no | no |

Slash shell parity comes from the CLI parser. The new v1.9 queue surface is MCP read-only; existing workflow-profile MCP tools still use the same service rules as the CLI, including request-review reviewer assignment and owner-only dependency override.

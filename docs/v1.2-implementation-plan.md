# Atlas Tasker v1.2 Implementation Plan

Atlas Tasker v1.2 builds on the shipped v1 local-first tracker and adds the shared contracts, policy model, lease model, review state, and service-layer foundations required for a real operator console.

## Scope anchor

- CLI stays canonical.
- `tracker shell` stays.
- `tracker tui` is additive later in the train.
- Markdown snapshots remain editable.
- JSONL remains append-only and authoritative for rebuild ordering.
- SQLite remains a rebuildable projection.
- New writes use schema version `2`.
- Existing v1 workspaces must keep loading without manual migration.

## PR train

- `PR-201` foundation / contracts
- `PR-202` claims / queues
- `PR-203` policy / review / dual gate
- `PR-204` hooks / doctor / inspect / repair
- `PR-205` templates / rollups / next
- `PR-206` TUI foundation
- `PR-207` TUI screens
- `PR-208` TUI write flows
- `PR-209` integrations
- `PR-211` hardening / release

## PR-201 deliverables

- v1.2 docs and decision log
- schema version `2` contracts
- review state and lease state contracts
- policy inheritance contracts
- new event types for claims/review/policy/template/owner-attention
- markdown codec compatibility for v1 and v1.2 snapshots
- SQLite migration for v1.2 ticket fields
- shared `internal/service` layer with action/query APIs
- minimum CLI read-path proof through shared query services
- backward-compat fixture coverage and service smoke tests

## Interface freeze for downstream PRs

`PR-201` freezes these shared shapes for the rest of the train:

- `contracts.ProjectDefaults`
- `contracts.TicketPolicy`
- `contracts.LeaseState`
- `contracts.ReviewState`

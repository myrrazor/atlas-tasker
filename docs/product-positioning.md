# Product Positioning

Atlas Tasker is a local-first, terminal-native task orchestration system for AI coding agents and humans.

It gives a small team the useful parts of Jira, a Kanban board, agent run tracking, review gates, evidence, handoffs, Git/worktree coordination, sync, signed artifacts, redaction, audit reports, backups, goal manifests, and MCP access without requiring a hosted server.

## Audience

- solo builders coordinating Codex, Claude Code, or other coding agents
- maintainers who want local, inspectable task state instead of a hosted tracker
- teams that need auditability, handoffs, and review gates around AI coding work
- release operators who need signed artifacts, backup/restore proof, and reproducible local evidence

## Product Promise

Atlas keeps task orchestration close to the code. Tickets, events, and evidence stay in the workspace; humans and agents use the same CLI, shell, TUI, Markdown, JSON, and MCP-backed surfaces; protected work goes through explicit governance instead of hidden side effects.

## What Not To Overclaim

- Atlas is not a hosted identity system.
- Atlas is not an OS sandbox.
- Atlas does not encrypt the workspace at rest.
- Atlas does not protect against a malicious local user with filesystem access.
- Atlas does not fully enforce external provider rules.
- Atlas does not make arbitrary MCP clients safe.

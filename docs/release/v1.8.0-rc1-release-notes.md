# Atlas Tasker v1.8.0-rc1 Release Notes

Status: local release candidate proof is green; public release sign-off is blocked on hosted proof.

## What Changed

- Reworked the public README and OSS metadata for a GitHub-facing project.
- Added a user-first docs system with install, quickstart, tutorial, concept, reference, troubleshooting, doctor/repair, and release-verification pages.
- Polished CLI, Markdown, and TUI output with shared render tokens, width-aware output, `NO_COLOR` behavior, plain output, ASCII fallback, and stronger empty states.
- Added agent workflow docs for Codex, Codex `/goal`, Claude Code, generic coding agents, and MCP.
- Added demo scripts, prompt packs, terminal transcripts, and screenshot-friendly examples.
- Added release preflight, version metadata, SBOM/vulnerability hooks, packaged rehearsal, and offline RC validation.
- Added final release evidence, launch checklist, CSO closeout, and a no-ship decision until hosted proof passes.

## Release Proof

Local proof passed:

- full tests and vet
- RC validator
- stability smoke
- packaged release rehearsal
- `govulncheck`
- CycloneDX SBOM generation
- proof-log leakage scan

Hosted proof did not pass:

- the current GitHub token could not read workflow permission settings
- no `v1.8.0-rc1` GitHub release exists yet

## Security Notes

Atlas can verify signed artifacts against trusted local keys, enforce Atlas application-level governance, redact Atlas-owned structured data, generate verifiable audit packets, and plan restore operations without side effects.

Atlas does not claim OS sandboxing, hosted identity, encrypted-at-rest storage, protection from malicious local filesystem users, formal DLP, full provider-rule enforcement, or full MCP client safety.

Read `docs/security-limitations.md` and `docs/release/public-release-gates.md` before treating any build as release-ready.

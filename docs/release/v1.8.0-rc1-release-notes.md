# Atlas Tasker v1.8.0-rc1 Release Notes

Status: local release candidate proof is green; public release sign-off is blocked on hosted proof.

## Who Should Use It

Use this branch for review and local RC testing only. Do not announce it as a public release until hosted assets, checksums, attestations, install smoke, and human docs/aesthetic sign-off are complete.

## Install

For now, build from source:

```bash
go build -o tracker ./cmd/tracker
./tracker version --json
```

After the hosted release exists, install from published assets:

```bash
VERSION=v1.8.0-rc1 BIN_DIR="$HOME/.local/bin" sh ./scripts/install.sh
```

The installer verifies checksums and GitHub artifact attestations by default.

## Verify

Before trusting a downloaded archive:

```bash
VERSION=v1.8.0-rc1 ./scripts/verify-release.sh ./tracker_1.8.0-rc1_darwin_arm64.tar.gz
```

Then compare `tracker version --json` against the release tag, commit, build date, and platform recorded in release evidence.

## Upgrade Notes

Back up the workspace before trying the RC:

```bash
tracker backup create --actor human:owner --reason "pre-rc backup"
tracker doctor --json
```

v1.8 is release polish on top of v1.7/v1.7.1. It should not require a manual data migration, but it is still blocked from public ship by hosted proof and carried-forward security review items.

## Security Notes

Atlas can verify signed artifacts against trusted local keys, enforce Atlas application-level governance, redact Atlas-owned structured data, generate verifiable audit packets, and plan restore operations without side effects.

Atlas does not claim OS sandboxing, hosted identity, encrypted-at-rest storage, protection from malicious local filesystem users, formal DLP, full provider-rule enforcement, or full MCP client safety.

Read `docs/security-limitations.md` and `docs/release/public-release-gates.md` before treating any build as release-ready.

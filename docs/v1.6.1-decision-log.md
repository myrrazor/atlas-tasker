## V161-008
- Date: 2026-05-05
- Question: How should Atlas expose MCP tools to coding agents without turning high-impact actions into model-controlled confirmations?
- Options Considered:
  - expose the old PR-310 thin adapter with `confirm: true`
  - expose all CLI commands through MCP with a server danger flag
  - expose profile-gated tools with out-of-band operation approvals for high-impact actions
- Chosen Option: profile-gated MCP tools with one-time operation approvals created outside MCP
- Why We Chose It: a boolean confirmation inside a tool call is not a safety boundary because the model can set it. Tool profiles keep the default surface small, and operation approvals bind a human-approved operation, target, actor, TTL, and single-use token before provider, sync, import, archive, compact, worktree cleanup, gate waive, or completion actions can run.
- Confidence: high
- Revisit Trigger: MCP clients provide a standard, verifiable human approval channel that Atlas can validate directly.
- Affected PRs/Files: PR-616, `internal/mcp`, `internal/cli/mcp.go`, `docs/mcp*.md`

## V161-009
- Date: 2026-05-06
- Question: How should MCP record post-approval execution failures and paginate grouped read surfaces?
- Options Considered:
  - only audit successful high-impact executions and pre-approval denials
  - audit failures after approval consumption with the consumed approval ID
  - keep a single cursor for grouped board/dashboard reads
  - use independent cursors keyed by board status or dashboard section
- Chosen Option: audit post-consumption failures with `execution_failed` and use keyed cursors for grouped MCP reads
- Why We Chose It: operation approvals are consumed before the service action starts, so a failed provider or service action still needs a chain-of-custody record. Board and dashboard sections have different lengths, so a single offset cursor cannot represent their independent pagination state without hiding shorter sections.
- Confidence: high
- Revisit Trigger: MCP gains a standard cursor object contract or Atlas moves grouped reads to separate per-section tools.
- Affected PRs/Files: PR-616, `internal/mcp/server.go`, `internal/mcp/tools.go`, `internal/mcp/output.go`, `docs/mcp*.md`

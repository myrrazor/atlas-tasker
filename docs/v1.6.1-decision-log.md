## V161-008
- Date: 2026-05-05
- Question: How should Atlas expose MCP tools to coding agents without turning high-impact actions into model-controlled confirmations?
- Options Considered:
  - expose the old PR-310 thin adapter with `confirm: true`
  - expose all CLI commands through MCP with a server danger flag
  - expose profile-gated tools with out-of-band operation approvals for high-impact actions
- Chosen Option: profile-gated MCP tools with one-time operation approvals created outside MCP
- Why We Chose It: a boolean confirmation inside a tool call is not a safety boundary because the model can set it. Tool profiles keep the default surface small, and operation approvals bind a human-approved operation, target, actor, TTL, and single-use token before provider, sync, import, archive, compact, worktree cleanup, gate waive, or completion actions can run.
- Confidence: high
- Revisit Trigger: MCP clients provide a standard, verifiable human approval channel that Atlas can validate directly.
- Affected PRs/Files: PR-616, `internal/mcp`, `internal/cli/mcp.go`, `docs/mcp*.md`
